git init	hace un seguimiento o respaldo de los archivos, !!solo se ejecuta al inicio del proyecto¡¡	
	
	git crea dos areas para seguir el archivo
		el área de ensayo	area temporal para hacer el seguimiento de los archivos
		el repsitorio local	donde realiza las instantaneas y sirven para recuperar los datos

git-add		sirve para decir a que archivos tiene que realizar un seguimiento

guit-commit	crea el repsitorio local crea el respaldo a partir del cual poder recuperar los archivos

git status -s	nos dice que archivos y directorios tenemos en la carpeta, en rojo si no someten a seguimiento

git add index.html	empieza a hacer un seguimiento del archivo y aparece dn color verde

git commit -m "comienzo del proyecto"	sirve para realizar un respaldo y guardar la instantanea con la descripcion para diferenciar las instantaneas

	la primera vez pedira username y una direccion de email y un usuario
git config --global user.username "nombre"  
git config --global user.email "email"	

git log --oneline	sirve para ver que copias de seguridad tenemos realizadas el numero de la izquierda se usa para recuperar esa copia de seguridad

git reset --hard b3f52c9	sirve para volver a la instantenea elegida con el codigo

git add .		sirve para añadir al area de seguimiento todos los archivos del proyecto a la vez

git commit -am "descripcion"	añade los todos los archivos a la zona de guardado y realiza una instantanes

	para modificar la descripcion tenemos que usar vim(editor de texto)
git commit --amend	sirve para modificar las descripciones de las copias de seguridad
	dentro pulsamos :i para poder editar,luego pulsamos cualquier tecla lo modificamos y guardamos con :wq
